## Learning Path

Week 4 has **5 topics** covering the foundational linear data structures that appear everywhere — in operating systems, compilers, parsers, and nearly every interview problem involving order and traversal. Work through them in the order below.

```
Linked List
    |
    |  Teaches pointer-based thinking. Every node holds data + a reference.
    |  This unlocks how dynamic memory structures actually work.
    v
Stack
    |
    |  LIFO structure built on top of what you know. Core to recursion,
    |  expression evaluation, and undo systems.
    v
Queue
    |
    |  FIFO structure. Appears in BFS, task scheduling, and rate limiters.
    |  Understand it here before using it in graphs next week.
    v
Deque
    |
    |  Generalises both Stack and Queue. Essential for sliding window
    |  problems and monotonic queue patterns.
    v
Backtracking
    |
    |  Recursive problem-solving technique. Explore all possibilities,
    |  prune dead ends. Foundation for Trees, Graphs, and DP later.
    v
Week 5

```

## Before You Move to Week 5

- Can you reverse a linked list without looking up the code?
- Can you implement a stack and queue from scratch using an array?
- Do you understand why a Deque is used for the sliding window maximum?
- Can you write the backtracking skeleton from memory and apply it to a new problem?
- Have you attempted all problems in the problem set?
